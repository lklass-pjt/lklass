# 07. Git And AI Usage

- Branch names should follow `feat/{feature-name}`, `fix/{bug-name}`, `docs/{topic}`, or `chore/{topic}` unless the user requests otherwise.
- Commit messages should use conventional commit style.
- Preserve unrelated user changes.
- Do not reset, checkout, clean, or revert files without explicit approval.
- README must briefly describe AI usage:
  - requirements organization
  - test case derivation
  - implementation assistance
  - debugging/review assistance
- Do not claim AI-generated work was manually written without review.
