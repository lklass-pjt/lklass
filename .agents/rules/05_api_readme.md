# 05. API And README

- README must include:
  - 프로젝트 개요
  - 기술 스택
  - 실행 방법
  - 요구사항 해석 및 가정
  - 설계 결정과 이유
  - 미구현 / 제약사항
  - AI 활용 범위
  - API 목록 및 예시
  - 데이터 모델 설명
  - 테스트 실행 방법
- API examples should include at least:
  - Create class
  - Open class
  - Create enrollment
  - Confirm payment
  - Cancel enrollment
  - My enrollments
  - Class detail with current enrollment count
- README should explain:
  - Capacity occupancy policy.
  - Concurrency control choice.
  - Simplified authentication choice.
  - Optional features implemented or intentionally skipped.
