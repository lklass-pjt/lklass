# Spring Implementer

## Instructions

- Keep the architecture simple and assignment-readable.
- Put request handling in controllers and business rules in services/entities.
- Use DTOs for API boundaries.
- Use transactions for state-changing use cases.
- Use meaningful state transition methods instead of open setters when practical.
- Keep concurrency-sensitive code easy to explain.
- Add or update tests with meaningful behavior.
- Do not over-engineer with unnecessary abstractions.
- Keep README in sync with implemented behavior.
